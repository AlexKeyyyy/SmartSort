# SmartSort: подробный отчет по нейросетевой части

## 1. Цель проекта

SmartSort — это система интеллектуальной сортировки файлов по категориям:
- Code
- Docs
- Finance
- Study
- Media
- Archives
- Logs
- Other

Практическая задача: автоматически определять тип файла по **имени файла** и **текстовому содержимому**, а затем при необходимости перемещать файл в соответствующую папку.

С точки зрения дисциплины «Нейронные сети» ключевая часть проекта — использование и дообучение трансформерной модели для многоклассовой классификации текстов.

## 2. Что использовано в текущей версии

В текущей рабочей версии используется transformer-модель:
- `distilbert/distilbert-base-multilingual-cased` (базовый вариант)

Также код поддерживает запуск модели из папки `models/smartsort_transformer/`.

Важно: в приложении сохранены rule-based правила по расширениям для некоторых классов (`Media`, `Archives`, `Code`, `Logs`) для повышения надежности на бинарных/служебных файлах.

## 3. Почему именно трансформер

### 3.1 Ограничения классических моделей

Классические ML-модели (например, TF-IDF + MLP) хорошо работают на простых шаблонах, но хуже переносятся на:
- неоднозначные формулировки,
- смешанные RU/EN тексты,
- вариативные контексты в длинных документах.

### 3.2 Преимущества трансформеров

Трансформеры учитывают контекст слова в предложении и лучше обобщают. Это важно для категорий вроде `Docs`, `Study`, `Finance`, где могут пересекаться термины.

Преимущества:
- контекстное представление текста;
- поддержка мультиязычности (RU+EN);
- удобный fine-tuning под задачу классификации.

## 4. Архитектура нейронной сети (как объяснять на защите)

В проекте используется архитектура `DistilBertForSequenceClassification`.

Упрощенно, цепочка выглядит так:
1. **Токенизация** входного текста.
2. **Эмбеддинги** токенов.
3. Несколько слоев self-attention + feed-forward (ядро трансформера).
4. Выходное представление (`[CLS]`-подобный вектор для классификации).
5. **Линейная классификационная голова** на 8 классов.
6. `softmax` -> вероятности классов.

### 4.1 Self-Attention (суть)

Self-attention вычисляет, какие токены важны друг для друга, и формирует контекстные векторы. Это позволяет модели учитывать зависимости в тексте, а не только частоты слов.

### 4.2 Классификационная голова

Последний слой — линейный `W*h + b`, где:
- `h` — вектор признаков текста,
- выход размерности 8 (по числу категорий).

Функция потерь при обучении — `CrossEntropyLoss`.

## 5. Формирование входа в нейросеть

Для каждого файла формируется единая строка формата:

`[FILE] <нормализованное имя> [CONTENT] <текст файла>`

### 5.1 Нормализация имени

Имя файла:
- приводится к нижнему регистру,
- `_` и `-` заменяются пробелами,
- расширение кодируется как отдельный токен вида `EXT_pdf`.

Это помогает сети учитывать сигналы от расширений и структуры имени.

### 5.2 Ограничение длины

Текст обрезается токенизатором до `max_length` (в текущей модели сохранено `128`), чтобы ограничить память и время инференса.

## 6. Данные и разметка

Обучение идет на `data/dataset.csv` с колонками:
- `filename`
- `content`
- `label`

Классы: 8 категорий SmartSort.

Датасет собирается из открытых источников + локальных seed-корпусов, в том числе русскоязычных, поэтому модель работает в мультиязычном режиме.

## 7. Процесс обучения

Скрипт: `src/train_transformer_model.py`

Шаги обучения:
1. Загрузка и проверка датасета.
2. Построение `label2id` / `id2label`.
3. Stratified split на train/validation.
4. Токенизация и формирование DataLoader.
5. Fine-tuning трансформера (AdamW + linear warmup scheduler).
6. Валидация по эпохам.
7. Early stopping по `macro_f1`.
8. Сохранение лучшей модели + метрик.

Сохраняются артефакты:
- модель и токенизатор: `models/smartsort_transformer/`
- метрики: `reports/model_eval/transformer/`

## 8. Инференс в приложении

В runtime-классе `SmartSortPredictor` реализованы два слоя логики:

1. **Rule-based приоритет** по расширению (для устойчивости):
- `Media`
- `Archives`
- `Code`
- `Logs`

2. **Нейросетевой инференс** для остальных файлов:
- извлечение текста;
- формирование входной строки;
- токенизация;
- прогон через transformer;
- выбор класса с максимальной вероятностью;
- если confidence ниже порога (`0.55`) -> `Other`.

Такое комбинирование повышает стабильность на бинарных файлах и логах, где текст может быть пустым/шумным.

## 9. Как объяснить выбор между DistilBERT / ruBERT / sentence-transformers

### DistilBERT
- Плюсы: быстрее, легче, удобен для демо на CPU.
- Минусы: обычно уступает в качестве более крупным моделям.

### ruBERT (например, `DeepPavlov/rubert-base-cased`)
- Плюсы: как правило, лучше на русском языке.
- Минусы: тяжелее, выше требования к ресурсам.

### sentence-transformers
- Обычно применяются для получения эмбеддингов и задач семантического поиска/кластеризации.
- Для прямой supervised-классификации в вашей задаче удобнее и прозрачнее `AutoModelForSequenceClassification`.

Итого для курсовой:
- baseline по скорости: DistilBERT;
- финальный quality-кандидат для RU: ruBERT;
- sentence-transformers можно упомянуть как альтернативную исследовательскую ветку (эмбеддинги + отдельный классификатор), но не как основной production-путь.

## 10. Текущий результат и интерпретация

В текущем сохраненном transformer-эксперименте (`reports/model_eval/transformer`) качество пока промежуточное, а не финальное.

Это нормально для этапа интеграции: модель уже подключена в приложение, но для защиты лучше сделать финальный прогон (желательно ruBERT) и зафиксировать улучшенные метрики.

## 11. Ограничения текущей версии

1. `max_length=128` может обрезать длинные документы.
2. Есть классы, которые сложнее разделяются (`Docs/Study/Finance`) при ограниченном контексте.
3. Баланс и качество исходных данных напрямую влияет на итоговый `macro_f1`.

## 14. Команды для запуска

Установка зависимостей:

```bash
python -m pip install -r requirements.txt
```

Запуск приложения (теперь использует `models/smartsort_transformer`):

```bash
python main.py
```

Тестовый инференс из CLI:

```bash
python -m src.predict_transformer \
  --filename "invoice_april_15.pdf" \
  --content "Сумма к оплате 15200 руб. НДС 20 процентов. Оплата в течение 5 банковских дней."
```

Переобучение transformer на текущем датасете:

```bash
python -m src.train_transformer_model
```

End-to-end пересборка датасета + обучение transformer:

```bash
python -m src.train_open_transformer_model --quick
```
